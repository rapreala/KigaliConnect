# KigaliConnect вҖ” Design Document

## 1. Architecture Overview

KigaliConnect follows **Clean Architecture** with three distinct layers. No layer may depend on a layer above it.

```text
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ           PRESENTATION LAYER                в”Ӯ
в”Ӯ  Screens вҶ’ BLoC (Events/States) вҶ’ Widgets  в”Ӯ
в”ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө
в”Ӯ              DATA LAYER                     в”Ӯ
в”Ӯ     Repositories вҶ’ Services (Firebase)     в”Ӯ
в”ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө
в”Ӯ             DOMAIN LAYER                    в”Ӯ
в”Ӯ        Models В· Enums В· Validators          в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

**Data flow (unidirectional):**

```text
UI Widget
  вҶ’ dispatches Event to BLoC
    вҶ’ BLoC calls Repository method
      вҶ’ Repository calls Service (Firebase)
        вҶ’ Service returns data
      вҶ’ Repository maps to domain model
    вҶ’ BLoC emits new State
  вҶ’ BlocBuilder rebuilds UI Widget
```

## 4. BLoC Design

### 4.1 AuthBloc

**Events:**

| Event | Payload | Description |
| --- | --- | --- |
| `AuthCheckRequested` | вҖ” | Start listening to Firebase auth state stream |
| `AuthSignInRequested` | email, password | Sign in with email/password |
| `AuthRegisterRequested` | email, password, displayName | Create Firebase user + Firestore profile |
| `AuthGoogleSignInRequested` | вҖ” | Sign in with Google OAuth |
| `AuthVerificationEmailRequested` | вҖ” | Resend verification email |
| `AuthSignOutRequested` | вҖ” | Sign out current user |

**States:**

| State | Fields | Description |
| --- | --- | --- |
| `AuthInitial` | вҖ” | App just started |
| `AuthLoading` | вҖ” | Any auth operation in progress |
| `AuthAuthenticated` | `UserProfile profile` | Logged in + email verified |
| `AuthEmailNotVerified` | вҖ” | Logged in but email not verified |
| `AuthUnauthenticated` | вҖ” | Logged out |
| `AuthFailure` | `String message` | Auth operation failed |
| `AuthVerificationEmailSent` | вҖ” | Verification email sent successfully |

**Flow diagram:**

```text
App Start
  вҶ’ AuthCheckRequested (subscribes to authStateChanges stream)
    вҶ’ Firebase user null        вҶ’ AuthUnauthenticated  вҶ’ LoginScreen
    вҶ’ Firebase user, unverified вҶ’ AuthEmailNotVerified вҶ’ EmailVerificationScreen
    вҶ’ Firebase user, verified   вҶ’ AuthAuthenticated    вҶ’ AppShell

LoginScreen
  вҶ’ Email/password form вҶ’ AuthSignInRequested
  вҶ’ Google button       вҶ’ AuthGoogleSignInRequested
    вҶ’ Both paths вҶ’ AuthAuthenticated вҶ’ AppShell
```

### 4.2 ListingsBloc

**Events:**

| Event | Payload | Description |
| --- | --- | --- |
| `LoadListings` | вҖ” | Subscribe to all listings stream |
| `LoadMyListings` | `String userId` | Load only current user's listings |
| `CreateListing` | `Listing listing` | Add new listing to Firestore |
| `UpdateListing` | `Listing listing` | Update existing listing |
| `DeleteListing` | `String listingId` | Remove listing from Firestore |
| `SearchListings` | `String query` | Filter by name (client-side) |
| `FilterByCategory` | `PlaceCategory? category` | Filter by category (null = all) |
| `ClearFilters` | вҖ” | Reset search and category filter |

**States:**

| State | Fields | Description |
| --- | --- | --- |
| `ListingsInitial` | вҖ” | Not yet loaded |
| `ListingsLoading` | вҖ” | Fetching from Firestore |
| `ListingsLoaded` | `allListings`, `filteredListings`, `searchQuery`, `selectedCategory` | Data ready |
| `ListingOperationSuccess` | `String message` | Create/update/delete succeeded |
| `ListingsError` | `String message` | Firestore operation failed |

**State transitions:**

```text
LoadListings вҶ’ ListingsLoading вҶ’ ListingsLoaded (stream subscription)
                                       вҶ‘
                         (auto-updates when Firestore changes)

CreateListing вҶ’ ListingsLoading вҶ’ ListingOperationSuccess вҶ’ [re-emits ListingsLoaded]
UpdateListing вҶ’ ListingsLoading вҶ’ ListingOperationSuccess вҶ’ [re-emits ListingsLoaded]
DeleteListing вҶ’ ListingsLoading вҶ’ ListingOperationSuccess вҶ’ [re-emits ListingsLoaded]

SearchListings вҶ’ ListingsLoaded (same data, filtered list updated)
FilterByCategory вҶ’ ListingsLoaded (same data, filtered list updated)
```

## 9. Firestore Security Rules

```text
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users can only read/write their own profile
    match /users/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }

    // Any authenticated user can read listings
    // Only the creator can update or delete
    match /listings/{listingId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null
                    && request.resource.data.createdBy == request.auth.uid;
      allow update, delete: if request.auth != null
                             && resource.data.createdBy == request.auth.uid;
    }
  }
}
```
